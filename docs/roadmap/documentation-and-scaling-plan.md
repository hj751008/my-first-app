# Documentation And Scaling Plan

## Goal
- Keep the project easy to resume while it grows from middle-1 into middle-2, middle-3, and high school content.

## Current Phase
- Middle-1 product flow is working.
- Validation harnesses exist for content, tutor behavior, preflight checks, and copyright review.
- Documentation structure is being split so future growth does not depend on one large root file.

## Next Documentation Milestones
1. Keep root docs short and current.
2. Start adding dated handoff files for major work sessions.
3. Move stable design rules into `docs/decisions/`.
4. Add grade-expansion planning docs before building middle-2 and high-school units.

## Next Product Milestones
1. Strengthen copyright harness reference coverage with plaintext extracts.
2. Decide whether copyright checks belong in preflight.
3. Define a grade-scalable unit schema and progression model.
4. Expand content and shared structure beyond middle-1.
